- 2026-06-25 (later) — Fix: Identifiers browser overlapping the Run output panel (same branch/PR #10).
  User hit it right after the flatten: with a run active, expanding the fund browser bled over the Run output fieldset. Systematic-debug → root cause: the expandable browsers (`#etfBox`/`#fundBox`) were `flex:none` with their own `38vh` bodies sitting *outside* any scroll region, while `.boxIds` has `overflow:visible` — so when the column split 50/50 with the (flex:1) Run output box, the browser content exceeded `.boxIds`'s share and painted over `.boxRun`. Every other variable-height area (right-column `.cols`, the `.chips`) already uses `flex:1; min-height:0; overflow:auto` to scroll within its box; the browsers skipped that pattern. Fix: wrap `#etfBox`+`#fundBox`+chips in one `.idscroll` region with that same pattern, and drop `.etfbody`'s inner `max-height:38vh`/`overflow` so there's a single scrollbar (4 edits, same file). Verified live via getBoundingClientRect (run active, BBVA expanded to 1704px): `.idscroll` scrolls internally and stays within `.boxIds` (616≤629), Run output sits below with a 23px gap, no visual overlap, zero console errors. Pre-existing layout bug the flatten merely exposed.
